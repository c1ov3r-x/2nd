# 📌 Portfolio 프로젝트 보고서

## 1. 프로젝트 제작 의도

사용자의 시선 흐름과 인터랙션을 고려한 웹 포트폴리오를 제작하는 것을 목표로 했다.

개발자로서의 역량을 보여주는 데 있어
정적인 정보 나열이 아니라 직접 경험하고 탐색할 수 있는 구조를 만들고자 했다.

또한 AI, 보안, 웹 개발 등 다양한 관심 분야를 하나의 흐름으로 연결하여
**“문제를 분석하고 해결하는 개발자”**라는 정체성을 명확하게 드러내는 데 집중했다.

---

## 2. 디자인 특징

* **다크 테마 기반 UI**
  → 시각적 피로를 줄이고, 개발자 포트폴리오에 어울리는 분위기 구성

* **네온 스타일 포인트 컬러**
  → 주요 요소(텍스트, 타임라인, hover 효과)에 강조를 주어 집중도 향상

* **타이포그래피 중심 레이아웃**
  → 불필요한 이미지 대신 텍스트와 구조로 정보 전달

* **스크롤 기반 페이지 구성**
  → 페이지 이동 없이 자연스럽게 콘텐츠를 탐색하도록 설계

* **타임라인 UI**
  → 경험을 시간 흐름에 따라 직관적으로 보여줌

---

## 3. 주요 기능

### 1) 스크롤 기반 네비게이션

* 스크롤 위치에 따라 헤더 메뉴(active)가 자동으로 변경
* 사용자가 현재 위치를 쉽게 인지 가능

### 2) 스킬 인터랙션 기능

* Language / Tools 버튼으로 카테고리 분리
* 클릭 시 해당 스킬 목록 동적 생성
* 스킬 클릭 시 상세 경험 설명 출력

### 3) 타임라인 애니메이션

* 스크롤 시 요소가 순차적으로 나타나는 효과
* 사용자에게 자연스러운 흐름 제공

### 4) 인터랙션 기반 UI 요소

* hover 시 카드 이동 및 네온 효과
* 클릭 가능한 타임라인 카드 (외부 링크 이동)

### 5) 반응형 디자인

* 화면 크기에 따라 레이아웃 자동 변경
* 모바일 환경에서도 사용 가능

---

## 4. 기술 스택

* **HTML**

  * 시맨틱 구조 기반 페이지 구성

* **CSS**

  * Flexbox / Grid를 활용한 레이아웃 설계
  * 애니메이션 및 네온 스타일 효과 구현

* **JavaScript**

  * DOM 조작을 통한 동적 UI 구현
  * 스크롤 이벤트 기반 인터랙션 처리
  * 데이터 기반 스킬 렌더링

---

## 5. 창의적 요소

사용자가 직접 탐색하며 이해하는 구조로 설계되었다.

특히 다음 요소에 차별성을 두었다.

* 스킬을 “목록”이 아니라 경험 중심 인터랙션 UI로 표현
* 타임라인을 활용한 경험의 흐름 시각화
* 네온 효과와 마이크로 인터랙션을 통한 몰입감 강화

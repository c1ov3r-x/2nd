# PS1 - 내신

## 문제 분석

문제를 처음 읽었을 때 가장 먼저 확인한 것은 등급을 결정하는 기준이었습니다.

학생들의 점수를 바탕으로 내신 등급을 계산하며, 
목표 등급을 받기 위해 필요한 최소 점수를 구해야 했습니다.

<등급 기준>

* 1등급 : 상위 10%
* 2등급 : 상위 34%
* 3등급 : 상위 66%
* 4등급 : 상위 90%
* 5등급 : 상위 100%

또한 동점자가 존재할 경우, 동점자 전원에게 가장 뒤의 등수를 부여한다는 조건이 있습니다.

처음에는 단순히 상위 X% 위치의 점수를 찾으면 된다고 생각했지만, 동점자 처리 규칙 때문에 단순 인덱스 계산만으로는 해결할 수 없다는 것을 알게 되었습니다.

예를 들어 점수가 다음과 같은 경우,

```text
100 95 95 90
```

95점을 받은 학생들은 2등과 3등을 차지하게 되는데, 문제 조건에 따라 둘 다 3등으로 처리됩니다.

즉, 일반적인 등수 계산이 아니라 동점자를 고려한 별도의 등수 계산 과정이 필요하다고 생각하였습니다.

---

## 문제 접근 과정

처음에는 학생 수와 등급 비율을 이용하여 목표 등급의 마지막 순위를 계산하였습니다.

예를 들어 학생 수가 100명이고 목표가 2등급이라면,

```text
100 × 0.34 = 34
```

이므로 34등까지가 2등급이 됩니다.

이후 학생들의 점수를 높은 순서대로 정렬하면 등수 계산이 쉬워질 것이라고 생각하였습니다.

따라서 다음과 같은 순서로 문제를 해결하기로 하였습니다.

1. 점수를 내림차순 정렬
2. 같은 점수를 받은 학생들을 찾음
3. 동점자 규칙에 따라 등수를 계산
4. 목표 등급에 포함되는 가장 낮은 점수를 찾음
5. 해당 점수를 출력

---

## 구현 방법

### 1. 점수 정렬

등수를 계산하기 위해 먼저 점수를 내림차순으로 정렬하였습니다.

```python
scores.sort(reverse=True)
```

이 과정을 통해 가장 높은 점수가 앞에 위치하게 되며 등수 계산이 쉬워집니다.

---

### 2. 동점자 처리

문제에서 가장 중요하다고 생각한 부분은 동점자 처리였습니다.

점수가 같은 학생들을 하나의 구간으로 묶기 위해 while문을 사용하였습니다.

예를 들어

```text
100 95 95 90
```

의 경우,

```text
95 95
```

를 하나의 그룹으로 인식합니다.

이후 해당 학생들에게 가장 뒤의 등수를 부여하였습니다.

<결과>

```text
점수 : 100 95 95 90
등수 : 1   3  3  4
```

이 과정이 없으면 문제 조건을 만족할 수 없다고 판단하였습니다.

---

### 3. 목표 등급 계산

등급별 비율을 딕셔너리에 저장하였습니다.

```python
grade_percent = {
    1: 0.10,
    2: 0.34,
    3: 0.66,
    4: 0.90,
    5: 1.00
}
```

그리고 학생 수와 곱하여 해당 등급의 마지막 등수를 계산하였습니다.

```python
max_rank = int(N * grade_percent[X])
```

---

### 4. 정답 탐색

계산된 등수 범위 안에 포함되는 학생들 중 가장 낮은 점수를 찾아 정답으로 사용하였습니다.

이 점수가 바로 목표 등급을 받기 위해 필요한 최소 점수라고 판단하였습니다.

또한 학생 수가 너무 적어 목표 등급 자체가 존재하지 않는 경우에는 문제 조건에 따라 0을 출력하도록 구현하였습니다.

---

## 시간 복잡도 분석

가장 많은 시간이 소요되는 과정은 점수 정렬입니다.

<정렬의 시간 복잡도>

```text
O(N log N)
```

이후 동점자 처리와 정답 탐색은 각각 한 번씩 전체 배열을 순회하므로

```text
O(N)
```

이다.

따라서 전체 알고리즘의 시간 복잡도는

```text
O(N log N)
```

으로 분석하였습니다.

## 느낀 점 및 한계점

문제를 자세히 읽어보니까 동점자 처리 조건이 있어서,
계산만 하는 문제가 아니라 등수를 따로 구해야 한다는 걸 알게 되었습니다.

문제를 풀면서 그냥 코드를 짜는 것보다
문제 조건을 제대로 이해하는 게 더 중요하다는 걸 느꼈습니다.

특히 동점자 같은 경우를 어떻게 처리하느냐에 따라
결과가 달라질 수 있어서 이런 부분을 더 신경 써야겠다고 생각하였습니다.

다만 이번 풀이는 기본적인 방식으로 구현했기 때문에
데이터가 많아지면 더 효율적인 방법이 필요할 것 같고,
다른 방식으로 더 개선해볼 수 있을 것 같습니다.
